Foire Aux Questions

J'utilise le port parallèle et j'ai obtenu l'erreur WARNING ! Windows fooled around with the LPT port bits !
La résolution de ce problème est décrite dans le chapitre sur la Page de L'interface
J'utilise VISTA et j'ai une erreur en utilisant le port parallèle
Sous VISTA l'accès au port parallèle nécessite d'enregistrer WxPic comme un programme privilégié. Ceci n'est nécessaire que pour WxPic V1.1. A partir de la V1.1.1, ceci est fait automatiquement. Note: quand WxPic a correctement été enregistré comme un programme privilégié, manuellement (V1.1) ou automatiquement (V1.1.1+) un petit bouclier apparaît sur l'icône du programme. Et lorsque WxPic est lancé une demande de confirmation est demandée par la sécurité (sauf si ce mécanisme a été désactivé sur le PC).
Une fois mon composant choisi j'obtiens l'erreur Can't open file 'device\<composant>.dev'
WxPic a besoind de la description détaillée des composants pour les programmer. Afin d'obtenir les description à jour WxPic utilise les fichiers fournis par Michochip dans l'atelier de développement MPLAB. Ces fichiers étant sous copyright de Microchip, ils ne peuvent être fournis dans l'installation de WxPic. Il faut donc les télécharger séparément sur la page de MPLAB et ensuite indiquer à WxPic où ces fichiers sont installés. A noter que à partir de V1.1.1 l'erreur ne s'affiche plus dans une boîte de dialogue mais dans la barre d'état et le focus est positionné sur le champ qui indique la localisation des fichier MPLAB. Son titre est en rouge tant que le fichier pour le composant n'a pas été trouvé.
